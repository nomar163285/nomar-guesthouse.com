# 個人宿 運営ナレッジベース

この基盤は、宿の運営情報を2層で管理するためのものです。

- **Layer 1：宿ナレッジベース**（Markdown） — 宿の知識・物語・手順。ホームページの文章、問い合わせ返信、各種掲載文を生成する元データ。
- **Layer 2：業務データ**（CSV、`data/`） — 予約・部屋・ゲスト・入金。稼働率や売上を分析する元データ。

## 各フォルダの役割

- `00_System/` — 呼称・表記ルールなど、すべての生成物が守るべき定義。最初に埋める。
- `10_Property/` — 宿の事実情報（所在地・部屋・設備・ルール）。HP/OTA掲載文の一次ソース。
- `20_Brand_Story/` — コンセプト・物語・文章トーン。HP/SNS文章の源泉。
- `30_Area_Guide/` — 周辺グルメ・観光・交通。ゲスト案内やHP「周辺情報」に使用。
- `40_FAQ/` — よくある質問。HPのFAQや返信下書きの素材。
- `50_Operations/` — 清掃・鍵・緊急時対応など、属人化しがちな運営手順。
- `60_Issues/` — トラブル事例ログ（1事例1ファイル）。
- `70_Guests/` — ゲスト傾向メモ（個人情報・取り扱い注意・最小限の記録）。
- `data/` — Layer 2：予約・部屋・ゲスト・入金のCSV。

## 基本の使い方

1. 宿の情報を `10_Property` 〜 `40_FAQ` に書き溜める。
2. 予約が入ったら `data/` の CSV に記録する（OTA予約も含む）。
3. 「ホームページの紹介文を作って」「今月の稼働率を出して」のように Claude Code に話しかける。

## 個人情報の取り扱い

`70_Guests/` と `data/guests.csv`、`data/reservations.csv` には個人情報が含まれます。
公開リポジトリには絶対に上げないでください（`.gitignore` で除外済み）。

## Discord運用の始め方

スマホのDiscordから一言送るだけでHPを更新できる仕組みです。常時起動しているこのPC側でBotを起動しておく必要があります。

1. このPCで `discord-bot` フォルダを開き、`start.bat` をダブルクリックする（「Botが起動しました」と表示されればOK）。
2. Discordアプリの「のま〜管理」サーバー → 「開発」チャンネルで、やってほしいことを日本語で送る（例：「FAQに○○を追加して」）。
3. Botが「受け取りました」→ 編集内容の報告 →「反映しました」と返事をくれる。
4. 数分後、Cloudflare Pagesが自動的に再公開し、サイトに反映される。

`start.bat` のウィンドウは閉じずに開いたままにしておいてください（閉じるとBotが停止します）。
`discord-bot/.env` にはBotの認証情報が入っているため、絶対に公開・共有しないこと（`.gitignore` で除外済み）。
